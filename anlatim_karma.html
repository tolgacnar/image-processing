01 Örnekleme (Sampling) - Uygulama

Örnekleme, görüntünün mekansal çözünürlüğünü azaltma işlemidir. INTER_NEAREST yönteminde, hedef koordinata en yakın olan orijinal piksel değeri doğrudan seçilir.

Adım s = 2 için (Yarı boyut):
Yeni[x, y] = Eski[x*2, y*2]
cv2.resize(img, (w//2, h//2), interpolation=cv2.INTER_NEAREST)

02 Örnekleme - Nereden Gelir?

📚 TEORİK ARKA PLAN

Nyquist-Shannon Örnekleme Teoremi

Bir sinyali doğru şekilde yeniden oluşturabilmek için, örnekleme frekansı sinyaldeki en yüksek frekansın en az 2 katı olmalıdır.

förnekleme ≥ 2 × fmax
Görüntülerde Anlam:
• Yüksek frekans = Keskin kenarlar, ince detaylar
• Düşük frekans = Düz alanlar, yumuşak geçişler
• Yetersiz örnekleme → Aliasing (merdiven etkisi)

İnterpolasyon Yöntemleri

Yöntem Hız Kalite
INTER_NEAREST En hızlı Pikselleşme
INTER_LINEAR Hızlı İyi
INTER_CUBIC Orta Çok iyi
INTER_LANCZOS4 Yavaş En iyi
Ne Zaman Hangisi?
• Büyütme: INTER_CUBIC veya LANCZOS4
• Küçültme: INTER_AREA (anti-aliasing)
• Gerçek zamanlı: INTER_LINEAR

03 Kuantalama (Quantization) - Uygulama

Kuantalama, piksellerin genlik (renk) derinliğini azaltır. Bit sayısı düştükçe renk geçişleri sertleşir.

Level = 2^bits (Seviye Sayısı)
Step = 256 / Level
Çıktı = (Girdi // Step) * Step

Örnek: 2-bit (4 renk seviyesi) için Step = 64. Her değer 0, 64, 128 veya 192'ye yuvarlanır.

levels = 2 ** bits step = 256 // levels quant = (gray // step) * step

8-bit Orijinal

45 120
180 250
→

2-bit Kuantalanmış

0 64
128 192

(45 // 64 = 0) * 64 = 0

Analog → Dijital Dönüşüm

Gerçek dünyada ışık yoğunluğu sürekli (analog) bir değerdir. Bilgisayarlar için bunu ayrık (dijital) seviyelere çevirmeliyiz.

Sürekli değer [0, 1] → Ayrık seviyeler {0, 1, 2, ..., L-1}
Bit Derinliği ve Seviye Sayısı:
• 1-bit → 2 seviye (siyah/beyaz)
• 4-bit → 16 seviye
• 8-bit → 256 seviye (standart)
• 16-bit → 65,536 seviye (HDR)

Her yuvarlama bir bilgi kaybı oluşturur. Bu hataya "kuantalama gürültüsü" denir.

05 Kernel (Filtre) Nedir?

"Bu pikselin değerini belirlerken, etrafındaki piksellere ne kadar önem vermeliyim?"

Kernel, bir pikselin çevresindeki komşularıyla nasıl etkileşime girdiğini tanımlayan matematiksel bir şablondur.

Kernel = Bakış Açısı

Her kernel, görüntüye farklı bir "soru" sorar:

  • Kenar algılama: "Merkez pikselle etrafı arasında fark var mı?"
  • Yumuşatma: "Etraftaki piksellerin ağırlıklı ortalaması nedir?"
  • Keskinleştirme: "Merkezi vurgula, çevreyi baskıla"
Konvolüsyon İşlemi:
Çıktı[x,y] = Σ Σ (Girdi[x+i, y+j] × Kernel[i,j])

Örnek: 3×3 Kernel

w₁ w₂ w₃
w₄ w₅ w₆
w₇ w₈ w₉

w₅ = merkez ağırlığı
diğerleri = komşu ağırlıkları

Sezgisel Anlam: Kernel, "hangi komşulara ne kadar güveniyorum?" sorusunun matematiksel karşılığıdır.

07 Gaussian Kernel - Nereden Gelir?

📚 TEORİK ARKA PLAN
"Yakın pikseller daha fazla etki eder" - Normal Dağılım İlkesi

Normal Dağılım (Bell Curve)

Gaussian kernel, istatistikteki normal dağılım fonksiyonundan gelir.

G(x,y) = (1/2πσ²) × e-(x²+y²)/2σ²
Fiziksel Anlam: Isı yayılımı denkleminin (heat diffusion equation) çözümüdür. Bir noktadaki ısı, çevresine yayılırken uzaklığa göre azalır.

Uzaklığa Göre Etki

  • Merkez (0,0) → %100 etki
  • 1 piksel ötedeki komşu → %60 etki
  • Köşe (±1,±1) → %37 etki

3×3 Gaussian (σ≈0.85)

1 2 1
2 4 2
1 2 1

× 1/16 ile normalize
Toplam = 16

Neden Bu Değerler?
• Merkez: e⁰ = 1 → ×4 ≈ 4
• Kenar: e⁻⁰·⁵ ≈ 0.6 → ×4 ≈ 2
• Köşe: e⁻¹ ≈ 0.37 → ×4 ≈ 1

09 Sobel Kenar Algılama - Uygulama

Sobel, pikseller arasındaki yoğunluk farkını (gradyan) hesaplar. Fark ne kadar büyükse kenar o kadar belirgindir.

Gx = [[-1, 0, 1], [-2, 0, 2], [-1, 0, 1]]
Gy = [[-1, -2, -1], [0, 0, 0], [1, 2, 1]]

Gradyan = √(Gx² + Gy²)
sobelx = cv2.Sobel(img, cv2.CV_64F, 1, 0, ksize=3) sobely = cv2.Sobel(img, cv2.CV_64F, 0, 1, ksize=3) magnitude = cv2.magnitude(sobelx, sobely)

10 Sobel Kernel - Nereden Gelir?

📚 TEORİK ARKA PLAN

Sorun: Saf Türev Gürültülüdür

Kenar algılamak için türev (değişim hızı) ölçmeliyiz.

En Basit Türev:
Girdi: [100, 105, 110]
Fark: [ 5, 5 ] ← Sağ - Sol
Problem: Görüntülerde gürültü vardır!
Gürültü de "yüksek fark" oluşturur → Yanlış kenar algılanır

Çözüm: Sobel = Türev + Yumuşatma

Sobel kernel'i iki işlemi birleştirir:

  • Yumuşatma: Dikey [1, 2, 1] → Gürültüyü azalt
  • Türev: Yatay [-1, 0, 1] → Farkı hesapla

Sobel X Kernel'inin Oluşumu

1
×
-1 0 1
=
-1 0 +1
-2 0 +2
-1 0 +1

Yumuşatma × Türev = Sobel

Neden ortadaki satır 2 kat?
Merkez piksele daha yakın satır, kenarın belirlenmesinde daha önemlidir. Bu, Gaussian ağırlıklandırmanın basitleştirilmiş halidir.

11 Morfolojik İşlemler - Uygulama

Dilation nesneleri kalınlaştırır, Erosion inceltir. Binary görüntüler üzerinde şekil işleme yapar.

Dilation: Y[x,y] = Max(Pencere ∩ Kernel)
Erosion: Y[x,y] = Min(Pencere ∩ Kernel)

Opening = Erosion → Dilation
Closing = Dilation → Erosion
kernel = np.ones((3,3), np.uint8) dilated = cv2.dilate(img, kernel) eroded = cv2.erode(img, kernel) opened = cv2.morphologyEx(img, cv2.MORPH_OPEN, kernel)

Structuring Element

Morfolojik işlemlerde kernel'e "structuring element" denir ve tamamen kullanıcı tarafından tasarlanır.

13 Emboss (Kabartma) Efekti

Emboss, görüntüye 3D derinlik hissi verir. Bir yöndeki pikselleri çıkarıp diğer yöndeki pikselleri toplar.

Kernel (Güneydoğu):
[[-2, -1, 0],
 [-1,  1, 1],
 [ 0,  1, 2]]

Sonuç + 128 (Nötr gri offset)
Neden +128?
Fark yoksa sonuç 0 olur. 128 ekleyerek nötr gri elde ederiz. Pozitif farklar açık, negatif farklar koyu görünür.
kernel = np.array([[-2,-1,0],[-1,1,1],[0,1,2]]) emboss = cv2.filter2D(gray, -1, kernel) result = np.clip(emboss + 128, 0, 255)
